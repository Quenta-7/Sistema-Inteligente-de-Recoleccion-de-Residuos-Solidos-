HU-001
AUTENTICACIÓN DE USUARIO
====================================================

Como usuario registrado,

Quiero iniciar sesión utilizando mis credenciales de acceso,

Para acceder de forma segura a las funcionalidades disponibles del sistema.

Prioridad: Alta
Actor: Usuario

Valor de Negocio:
Permite el acceso seguro a la plataforma y protege la información de los usuarios.

CRITERIOS DE ACEPTACIÓN

Funcionales
- El usuario debe poder ingresar correo electrónico y contraseña.
- El sistema debe validar campos obligatorios.
- El sistema debe verificar que las credenciales correspondan a un usuario registrado.
- Si la autenticación es exitosa, el usuario debe acceder al Dashboard.
- Si la autenticación falla, el sistema debe mostrar un mensaje adecuado.

Técnicos
- Las contraseñas deben almacenarse utilizando BCrypt.
- La autenticación debe generar un token JWT.
- La sesión debe mantenerse mediante token seguro.
- Todas las comunicaciones deben realizarse mediante HTTPS.

Seguridad
- El sistema debe proteger las credenciales almacenadas.
- Debe prevenir ataques de fuerza bruta mediante limitación de intentos.

SUBTASKS
- Diseñar pantalla Login.
- Implementar API de autenticación.
- Implementar cifrado BCrypt.
- Implementar JWT.
- Implementar validaciones.
- Realizar pruebas de autenticación.

HU-002
REGISTRO DE CIUDADANO MEDIANTE DNI
====================================================

Como ciudadano que aún no posee una cuenta,

Quiero registrarme utilizando mi número de DNI,

Para crear una cuenta y acceder a los servicios digitales de la plataforma.

Prioridad: Alta
Actor: Ciudadano

CRITERIOS DE ACEPTACIÓN

Funcionales
- El formulario debe solicitar:
  * DNI
  * Distrito
  * Correo electrónico
  * Contraseña
  * Confirmación de contraseña
- El DNI debe contener exactamente 8 dígitos.
- El sistema debe consultar automáticamente los nombres y apellidos.
- El DNI no debe estar previamente registrado.
- El correo no debe estar previamente registrado.

Técnicos
- Integración con servicio de consulta de DNI.
- Contraseña almacenada mediante BCrypt.

Seguridad
- Validación de formato y unicidad.
- Comunicación mediante HTTPS.

Reglas de Negocio
- Contraseña mínima de 8 caracteres.
- Al menos una mayúscula.
- Al menos una minúscula.
- Al menos un número.
- Al menos un carácter especial.

SUBTASKS
- Diseñar formulario de registro.
- Integrar consulta de DNI.
- Implementar validaciones.
- Implementar registro de usuario.
- Implementar cifrado de contraseña.
- Realizar pruebas.
